🌐 How Data Updates Work

Every time someone opens the site, it fetches the latest data.js file directly from your GitHub repo. So all you need to do is:

1. Edit in Edit Mode → 2. Click ↓ Save JSON → 3. Upload to GitHub → ✓ Everyone sees it
✓ No server needed ✓ No tokens or API keys ✓ Nothing can break ✓ Free forever

📤 How to Upload data.js to GitHub

Option A — GitHub website (easiest):
Go to your repo → public/data/data.js → click the pencil ✏️ icon → click the upload icon → choose your downloaded file → click Commit changes.

Option B — GitHub Desktop: Copy the downloaded file into your local repo folder, then commit and push in GitHub Desktop.

Option C — Terminal:

cp ~/Downloads/data.js /path/to/your/repo/public/data/data.js
git add public/data/data.js
git commit -m "Update data"
git push
Note on Instagram auto-fetch: Public visitors don't see live-fetch controls — they're admin-only, since each run uses the site owner's Apify quota. Manual entry via Edit Mode or Bulk Import always works as a fallback. Live refresh is available in the ↓ Export panel.
